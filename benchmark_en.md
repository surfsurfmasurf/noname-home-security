# Anomaly Detection Model Benchmark Report

## Overview

This report compares **5 anomaly detection approaches** on synthetic API security traffic data generated by the Noname Home Security system. The goal is to evaluate our unsupervised ensemble model against top-performing supervised and unsupervised methods from Kaggle and HuggingFace.

## Models Tested

| # | Model | Type | Origin | Description |
|---|-------|------|--------|-------------|
| 1 | **IF + AE + Signature Ensemble** | Unsupervised | Ours | Isolation Forest (40%) + Autoencoder (40%) + Signature rules (20%) |
| 2 | **Random Forest** | Supervised | Kaggle (NSL-KDD #1) | 200 trees, max depth 20, StandardScaler preprocessing |
| 3 | **XGBoost** | Supervised | Kaggle (UNSW-NB15 #1) | 200 estimators, max depth 8, learning rate 0.1 |
| 4 | **Deep Neural Network (MLP)** | Supervised | HuggingFace | 3-layer MLP (128→64→32), BatchNorm, Dropout, weighted BCE loss |
| 5 | **One-Class SVM** | Unsupervised | HuggingFace | RBF kernel, nu=0.1, trained on normal data only |

## Dataset

- **Generator**: Synthetic API traffic generator with 5 device profiles and 6 attack types
- **Total samples**: 10,000
- **Train/Test split**: 70% / 30% (stratified)

## Methodology

### Training Conditions

- **Supervised models** (Random Forest, XGBoost, DNN): Trained with labeled data (normal=0, attack=1). These models know which samples are attacks during training.
- **Unsupervised models** (Our Ensemble, One-Class SVM): Trained without labels. Our ensemble trains on all data learning density patterns. One-Class SVM trains on normal data only, learning the boundary of normality.

### Evaluation

- All models evaluated on the same held-out test set (3,000 samples, 588 attacks, 2,412 normal)
- Metrics: Accuracy, Precision, Recall (TPR), F1 Score, AUC-ROC
- Our ensemble uses threshold=50 (out of 0-100 score range) for binary classification

### Environment

- Python 3.11, scikit-learn, PyTorch (CPU), XGBoost 3.2
- Hardware: Intel 12th Gen (12 threads), 32GB RAM, no GPU
- All benchmarks run sequentially on the same machine

## Analysis

### Why Supervised Models Dominate

Random Forest, XGBoost, and DNN achieve near-perfect scores because they are **supervised classifiers** trained with labeled data. They learn the exact decision boundary between normal and attack traffic. In this synthetic dataset, attack patterns are distinct enough for these models to achieve >99.9% accuracy.

### Our Model's Strengths and Weaknesses

**Low Recall (10.71%)**: Our ensemble misses most attacks. This is because:
- Isolation Forest learns data density, not attack labels — some attacks have features similar to normal traffic
- The Autoencoder reconstruction error may not diverge enough for subtle attacks
- Threshold=50 is too high for this data distribution

**High Precision (98.44%)**: When our model does flag something, it's almost always correct (only 1 false positive out of 2,412 normal samples).

**High AUC (0.9395)**: The model's scoring function can actually rank attacks higher than normal traffic. A lower threshold would dramatically improve recall at the cost of more false positives.

### One-Class SVM Comparison

One-Class SVM is also unsupervised but achieves 100% recall with 73% precision. The key difference: it's trained **only on normal data**, while our ensemble trains on all data. This gives OC-SVM a cleaner model of normality.

### Fairness of Comparison

This benchmark is **not an apples-to-apples comparison**:

| Aspect | Supervised (RF, XGB, DNN) | Unsupervised (Ours, OC-SVM) |
|--------|:-------------------------:|:----------------------------:|
| Requires labels | Yes | No |
| Works on new attack types | Limited | Yes |
| Real-world applicability | Need labeled dataset | Works immediately |
| Overfitting risk on synthetic data | High | Low |

In production, labels are rarely available for new/unknown attack types. Unsupervised approaches are essential for **zero-day detection**.

## Recommendations

1. **Lower the alert threshold** from 50 to 30 to improve recall
2. **Train Autoencoder on normal data only** (like OC-SVM) for cleaner anomaly boundary
3. **Hybrid approach**: Use our ensemble for initial scoring + a supervised model fine-tuned on confirmed alerts
4. **Feature engineering**: Add more discriminative features (e.g., request entropy, n-gram patterns)
5. **Increase training data**: More samples improve unsupervised model density estimation

## Conclusion

Supervised models (RF, XGBoost, DNN) achieve near-perfect detection on synthetic labeled data, as expected. However, our unsupervised ensemble trades recall for the ability to detect unknown attack types without labeled training data — a critical requirement for real-world API security monitoring. The high AUC (0.94) confirms the model has strong ranking ability and can be improved with threshold tuning and architectural refinements.
