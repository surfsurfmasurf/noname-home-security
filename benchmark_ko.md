# 이상 탐지 모델 벤치마크 보고서

## 개요

Noname Home Security 시스템에서 생성한 합성 API 보안 트래픽 데이터를 기반으로 **5가지 이상 탐지 모델**을 비교한 보고서입니다. 우리의 비지도학습 앙상블 모델을 Kaggle과 HuggingFace의 상위 모델들과 비교합니다.

## 테스트 모델

| # | 모델 | 학습 방식 | 출처 | 설명 |
|---|------|----------|------|------|
| 1 | **IF + AE + Signature 앙상블** | 비지도학습 | 자체 개발 | Isolation Forest (40%) + Autoencoder (40%) + 시그니처 규칙 (20%) |
| 2 | **Random Forest** | 지도학습 | Kaggle (NSL-KDD 1위) | 200개 트리, max depth 20, StandardScaler 전처리 |
| 3 | **XGBoost** | 지도학습 | Kaggle (UNSW-NB15 1위) | 200 estimators, max depth 8, learning rate 0.1 |
| 4 | **Deep Neural Network (MLP)** | 지도학습 | HuggingFace | 3층 MLP (128→64→32), BatchNorm, Dropout, 가중 BCE 손실 |
| 5 | **One-Class SVM** | 비지도학습 | HuggingFace | RBF 커널, nu=0.1, 정상 데이터만으로 학습 |

## 데이터셋

- **생성기**: 5개 디바이스 프로필과 6개 공격 유형을 포함한 합성 API 트래픽 생성기
- **총 샘플 수**: 10,000
- **학습/테스트 분할**: 70% / 30% (계층화 분할)
- **라벨 분포**:

## 방법론

### 학습 조건

- **지도학습 모델** (Random Forest, XGBoost, DNN): 라벨 데이터(normal=0, attack=1)로 학습. 학습 시 어떤 샘플이 공격인지 알고 있음.
- **비지도학습 모델** (우리 앙상블, One-Class SVM): 라벨 없이 학습. 우리 앙상블은 전체 데이터의 밀도 패턴을 학습. One-Class SVM은 정상 데이터만으로 정상의 경계를 학습.

### 평가

- 모든 모델을 동일한 테스트 세트로 평가 (3,000 샘플: 공격 588, 정상 2,412)
- 지표: 정확도, 정밀도, 재현율(TPR), F1 점수, AUC-ROC
- 우리 앙상블은 0-100점 범위에서 threshold=50 기준으로 이진 분류

### 환경

- Python 3.11, scikit-learn, PyTorch (CPU), XGBoost 3.2
- 하드웨어: Intel 12세대 (12 스레드), 32GB RAM, GPU 미사용
- 모든 벤치마크를 동일 머신에서 순차 실행

## 분석

### 지도학습 모델이 압도적인 이유

Random Forest, XGBoost, DNN은 **라벨이 있는 데이터**로 학습하는 지도학습 분류기입니다. 정상과 공격 사이의 정확한 결정 경계를 학습하기 때문에 합성 데이터셋에서 거의 완벽한 점수를 달성합니다.

### 우리 모델의 강점과 약점

**낮은 재현율 (10.71%)**: 대부분의 공격을 놓침. 이유:
- Isolation Forest는 데이터 밀도를 학습하지, 공격 라벨을 학습하지 않음 — 일부 공격은 정상 트래픽과 유사한 특성을 가짐
- Autoencoder의 재구성 오류가 미묘한 공격에 대해 충분히 크지 않음
- Threshold=50이 이 데이터 분포에 너무 높음

**높은 정밀도 (98.44%)**: 모델이 공격으로 판단했을 때 거의 항상 맞음 (정상 2,412개 중 오탐 1개뿐).

**높은 AUC (0.9395)**: 모델의 스코어링 함수는 실제로 공격을 정상보다 높은 점수로 매길 수 있음. Threshold를 낮추면 재현율이 크게 개선됨.

### One-Class SVM과의 비교

One-Class SVM도 비지도학습이지만 재현율 100%, 정밀도 73%를 달성. 핵심 차이점: **정상 데이터만으로 학습**하여 정상의 경계를 더 명확히 학습.

### 비교의 공정성

이 벤치마크는 **동일 조건 비교가 아닙니다**:

| 항목 | 지도학습 (RF, XGB, DNN) | 비지도학습 (우리 모델, OC-SVM) |
|------|:----------------------:|:----------------------------:|
| 라벨 필요 여부 | 필요 | 불필요 |
| 새로운 공격 유형 탐지 | 제한적 | 가능 |
| 실환경 적용 가능성 | 라벨 데이터셋 필요 | 즉시 적용 가능 |
| 합성 데이터 과적합 위험 | 높음 | 낮음 |

실제 운영 환경에서는 새로운/미지의 공격 유형에 대한 라벨이 존재하지 않습니다. 비지도학습 방식은 **제로데이 탐지**에 필수적입니다.

## 개선 방향

1. **알림 threshold 하향**: 50 → 30으로 조정하여 재현율 향상
2. **정상 데이터만으로 Autoencoder 학습**: OC-SVM처럼 더 명확한 이상 경계 형성
3. **하이브리드 접근**: 우리 앙상블로 초기 스코어링 + 확인된 알림으로 지도학습 모델 파인튜닝
4. **특성 엔지니어링**: 더 변별력 있는 특성 추가 (예: 요청 엔트로피, n-gram 패턴)
5. **학습 데이터 증가**: 더 많은 샘플로 비지도학습 밀도 추정 개선

## 결론

지도학습 모델(RF, XGBoost, DNN)은 합성 라벨 데이터에서 거의 완벽한 탐지율을 달성합니다. 이는 예상된 결과입니다. 반면 우리의 비지도학습 앙상블은 재현율이 낮지만, **라벨 없이 미지의 공격 유형을 탐지할 수 있다**는 실제 환경에서 필수적인 강점을 가집니다. 높은 AUC(0.94)는 모델의 순위 매김 능력이 우수함을 확인해주며, threshold 조정과 아키텍처 개선으로 충분히 개선 가능합니다.
